Dashboard Summary

Filtering Capabilities

The dashboard provides robust filtering capabilities to analyze benchmark results effectively. Users can refine their view based on specific criteria:

Date Range:

Allowing the user to select a start and end date. This enables time-based analysis and trend identification. Users can analyze performance over specific periods, such as daily, weekly, or monthly comparisons.

Run ID:

Filter results based on specific Run IDs, which represent unique identifiers for each benchmark run. This allows for isolating and comparing the performance of individual runs. Users can analyze specific runs in detail or compare different runs to understand the impact of configuration changes or other factors.

Index Type:

Filter by the type of vector index used (e.g., HNSW, IVF, etc.). This allows for a direct comparison of different indexing algorithms and their performance characteristics. Users can assess which index type is most suitable for their specific data and query patterns. It also allows for isolation of tests that utilize specific index parameters.

Key Performance Indicators (KPIs) and Charts

The dashboard currently visualizes key performance indicators through a single image containing four primary charts:

This image displays the following metrics: QPS (Queries Per Second), Latency p50 (50th Percentile Latency), Latency p95 (95th Percentile Latency), and Index Build Time.

These charts, combined with the filtering options, allow users to analyze performance trends, identify bottlenecks, and optimize their vector search configurations effectively. The filtering allows the user to view the charts based on date range, Run ID, and index type.
